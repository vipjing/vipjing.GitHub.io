TiDB 5.0 正式发布

服务核心场景的企业级分布式数据库

经过近一年紧锣密鼓的开发和打磨，TiDB 5.0 具备金融行业核心业务场景的服务能力，成为真正服务核心场景的企业级分布式数据库。

TiDB 5.0 性能与稳定性得到显著提升，从而拥有更强大的 OLTP 金融级核心场景服务能力；在原有 HTAP 引擎 TiFlash 基础上引入 MPP 架构，使得众多企业的实时/交互式 BI 成为现实，为高成长企业和数字化创新场景提供了一栈式的数据服务底座，加速带动 HTAP 进入更多大型企业的数字化场景。

TiDB 5.0 最新特性
多项性能指标实现突破
通过开启聚簇索引、异步提交事务、MPP 架构提升性能

聚簇索引：开启后，TiDB 在 Sysbench 测试中，Read Write 性能提升达 58.1%;

异步提交事务：开启后，平均延迟由 12.04 ms 降到 7.01ms ，降幅达 41.7%;

MPP 引擎：MPP 引擎总体性能是 Greenplum 6.15.0 与 Apache Spark 3.1.1 的 2-3 倍，部分查询可达 8 倍。

探索更多功能
稳定性大幅提升
在 TPC-C OLTP 基准测试中，衡量 TPC-C tpm-C 抖动标准差的值小于等于 2%

优化资源调度：优化 TiDB 调度过程中对 I/O、网络、CPU、内存等资源的占用；

优化 SQL 语句：优化 SQL 语句，确保 TiDB 新版本稳定性；

优化升级计划：新增自动捕获并绑定查询计划的功能。

探索更多功能
全方位提升易用性
提供更加简易的运维体验，全方位提升 TiDB 易用性

更精准的性能排查：通过 EXPLAIN 可查看日志、监控等详细信息；

更高效的集群部署：新版本 TiUP 提供业务无感知的升级体验，升级过程中性能抖动控制在 10~30 秒；

更全面的 SQL 调优：支持不可见索引 (Invisible Indexes)。

探索更多功能
丰富数据生态
增加多个数据迁移，数据导入，数据共享组件，方便用户在异构环境使用 TiDB

数据迁移：支持 S3 存储服务作为数据迁移的中间转存介质，支持将 Aurora 快照数据直接初始化 TiDB 中；

数据导入：TiDB Lightning 针对 DBaaS AWS T1.standard 配置（及其等同配置）的 TiDB 集群数据导入进行优化，性能提升 40%；

数据共享：TiCDC 集成 Kafka Connect ，并支持在多套 TiDB 集群之间相互复制数据。

新增多项企业级特性，带动企业用户加速推进数字化转型与升级

事务方面：悲观模式下，确保事务提交成功性；

高可用与容灾：引入 Raft Joint Consensus 算法，提升 Region 成员变更时的可用性；

安全合规：错误信息与日志信息脱敏处理，避免敏感信息泄露。
